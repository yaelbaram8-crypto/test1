# Family Smart Shopping App (Inspired by "Sally") 🛒

המטרה היא לבנות אפליקציה משפחתית חכמה לניהול רשימות קניות, השואבת השראה מבוט הוואטסאפ הפופולרי "סאלי" (ניהול רשימות שיתופי ופשוט). בנוסף, האפליקציה תכלול יכולות מתקדמות של מעקב אחר היסטוריית רכישות, ניתוח חשבוניות (צילום קבלות), והמלצות קנייה חכמות המבוססות על מחירים והעדפות עבר.

## תכונות הליבה (Core Features)

1. **ניהול רשימות שיתופי (בדומה ל"סאלי")**
   * יצירת רשימת קניות משפחתית מסונכרנת בזמן אמת.
   * הוספה מהירה של פריטים (השלמה אוטומטית, אפשרות להקלדה חופשית).
   * סידור אוטומטי של מוצרים לפי מחלקות בסופר (ירקות, חלבי, בשר וכו') לחוויית קנייה מהירה.
   * סימון מוצרים שנרכשו בלחיצה.

2. **זיכרון קניות והיסטוריה (Purchase Memory)**
   * מסד נתונים שישמור את כל היסטוריית הקניות של המשפחה.
   * יכולת חיפוש מהירה: "מתי בפעם האחרונה קנינו קפה?", "כמה עלה לנו חלב פעם קודמת?".
   * צפייה בסטטיסטיקות קנייה כלליות למעקב אחר הוצאות.

3. **סריקת קבלות חכמה (Receipt OCR)**
   * העלאת תמונה של חשבונית/קבלה מהסופר לאפליקציה.
   * קריאת טקסט אוטומטית (OCR) לחילוץ: 
     * שמות הפריטים שנקנו.
     * המחיר של פריט.
     * סך כל החשבונית ותאריך הרכישה.
   * קישור אוטומטי בין הפריטים שזוהו בקבלה לבין הפריטים שהיו ברשימה, למעקב סגור.

4. **מנוע המלצות ותזכורות (Smart Recommendations)**
   * **המלצות חידוש מלאי**: התראה אוטומטית שמוסיפה מוצר לרשימה על סמך מחזוריות (למשל: "אתם קונים חלב כל 4 ימים, שכחתם להוסיף").
   * **השוואת מחירים היסטורית**: הצגת המחיר הממוצע או המחיר האחרון ששולם על מוצר בעת ההוספה שלו לרשימה.
   * התראות חכמות: "בסופר א' שילמתם על המוצר הזה פחות מבסופר ב'".

## Proposed Changes / Architecture

### 1. Front-End (ממשק משתמש)
אנחנו ניצור אפליקציית Web מודרנית ורספונסיבית שתיראה מעולה במסכי טלפון (Mobile First). 
* **טכנולוגיות**: React / Next.js / Vite - עם עיצוב זורם ונעים המאפשר לסמן מוצרים באצבע במהלך הקנייה.

### 2. Back-End וניהול הנתונים (מסד נתונים)
נצטרך לשמור את הנתונים כך שכל המשפחה תוכל לראות את העדכונים מיד.
* **טכנולוגיות**: Firebase או Supabase לניהול משתמשים (משפחות) ומסד נתונים בזמן אמת (Realtime Database). 

### 3. מנוע סריקת קבלות (AI / OCR)
בניית Endpoint שידע לקבל תמונה ולחלץ ממנה מידע מובנה למערכת.
* **טכנולוגיות מועדפות**: Google Cloud Vision API או OpenAI gpt-4-vision. מכיוון שהקבלות בסופר הן ב**עברית** ולעיתים קשות לקריאה, שימוש ב-AI חזק יתן את דיוק הנתונים הכי טוב.
